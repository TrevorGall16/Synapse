3 · Dirty State Indicator — E2E Test Hook Placement

The dirty-state sentinel is invisible to users but always in the DOM so Playwright can query it. It lives in AppBootstrap — rendered at root layout level, screen-reader only. The mockup below shows where it lives in the component tree and what attributes tests use.

Component anatomy
<span data-testid="dirty-state-indicator"
     data-dirty="false"
     data-flushing="false"
     class="sr-only" aria-hidden />
Always in DOM — even when dirty=false
Playwright queries: [data-dirty="true"] to assert unsaved changes
Playwright queries: [data-flushing="true"] to assert overlay shown

E2E test contract: The data-testid="dirty-state-indicator" element is always present regardless of dirty state. Tests use attribute selectors — e.g., [data-dirty="true"] — to observe state transitions without depending on element visibility or the overlay appearing. The save-barrier overlay (data-testid="save-barrier-overlay") appears separately only when isFlushing=true.
